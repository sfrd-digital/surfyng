PROMPT PRINCIPAL (colar no Manus para começar):

---

Quero que você construa um aplicativo mobile chamado SURFYNG — um app de surf que indica a melhor praia para surfar agora, baseado na localização GPS do usuário + condições de vento e ondulação em tempo real.

Você vai criar:
1. App mobile React Native com Expo (iOS + Android)
2. Backend Node.js + Express com API REST
3. Banco de dados PostgreSQL
4. Cache Redis (condições meteorológicas)
5. Deploy no VPS Hostinger (Ubuntu 22.04)

---

STACK TÉCNICA OBRIGATÓRIA:
- Frontend mobile: React Native + Expo SDK 51+
- Backend: Node.js 20 + Express 4
- Banco de dados: PostgreSQL 15
- Cache: Redis 7
- Autenticação: Firebase Authentication (Google + Apple + Facebook login)
- Notificações push: Firebase Cloud Messaging (FCM)
- Pagamento: Stripe (modelo app-to-web via Stripe Checkout — sem IAP nativo)
- Hospedagem backend: VPS Hostinger (Ubuntu 22.04, Nginx, PM2)
- Mapas: React Native Maps + Google Maps SDK
- HTTP client: Axios

---

CONCEITO DO APP:

O Surfyng resolve um problema real: surfistas (especialmente iniciantes) não sabem interpretar sites de previsão de ondas. O app usa GPS + API do Windguru para buscar condições em tempo real e indica, de forma simples, a praia mais próxima com as melhores condições para o nível do surfista.

Fluxo principal:
1. Usuário abre o app
2. GPS captura localização atual
3. Backend busca praias num raio de 50km (banco de dados)
4. Para cada praia, busca condições via Windguru API (cache Redis 30min)
5. Algoritmo calcula score 0-10 por praia
6. App exibe: praia indicada + distância + condições + temperatura da água + roupa recomendada

ALGORITMO DE MATCHING (lógica central do app):

function calculateScore(beach, conditions):
  score = 0

  // Vento (4 pontos)
  if conditions.wind_direction in beach.wind_directions:
    score += 4.0
  else if conditions.wind_direction is adjacent to beach.wind_directions:
    score += 2.0

  // Ondulação (4 pontos)
  if conditions.swell_direction in beach.swell_directions:
    score += 4.0
  else if conditions.swell_direction is adjacent:
    score += 2.0

  // Tamanho (2 pontos)
  wave_feet = conditions.swell_height * 3.28
  if wave_feet >= beach.min_size_feet AND wave_feet <= beach.max_size_feet:
    score += 2.0
  else if wave_feet >= beach.min_size_feet * 0.7:
    score += 1.0

  // Bônus período
  if conditions.swell_period >= 10:
    score += 0.5
  else if conditions.swell_period >= 8:
    score += 0.3

  return min(score, 10.0)

Filtro por nível do usuário:
- beginner: dificuldade low ou medium_low
- intermediate: dificuldade low, medium_low ou medium
- advanced: todas as dificuldades

Ordenação final: score DESC, distância ASC

INTEGRAÇÃO WINDGURU:

URL base: https://www.windguru.cz/int/iapi.php
Parâmetros: uid={station_id}&q=forecast&format=json

Dados a extrair da resposta:
- WINDSPD (velocidade do vento em nós)
- WINDDIRNAME (direção do vento)
- SHWW (altura da ondulação em metros)
- DIRPW (direção da ondulação)
- PERPW (período da ondulação em segundos)
- TMP (temperatura do ar)
- SST (temperatura da superfície do mar — se disponível)

Cache no Redis: 30 minutos por estação

Fallback: se Windguru indisponível, usar último cache válido ou exibir mensagem de manutenção

PAGAMENTO — STRIPE APP-TO-WEB:

Modelo: o usuário clica em "Assinar Pro" no app → app abre browser com URL do Stripe Checkout → usuário paga → Stripe redireciona para deeplink do app → app verifica assinatura ativa via backend

Implementar:
1. POST /api/v1/subscriptions/checkout
   - Criar ou recuperar stripe_customer_id
   - Criar sessão de checkout com price_id correto
   - success_url: surfyng://payment-success (deeplink)
   - cancel_url: surfyng://payment-cancel (deeplink)
   - Retornar URL do checkout

2. Webhook POST /api/v1/webhooks/stripe
   - checkout.session.completed → atualizar plano do usuário
   - customer.subscription.updated → sincronizar status
   - customer.subscription.deleted → rebaixar para free
   - invoice.payment_failed → notificar usuário

3. No app React Native:
   - Usar Linking.openURL(checkoutUrl) para abrir o browser
   - Configurar deep link surfyng:// para capturar retorno
   - Ao retornar, chamar GET /subscriptions/current para confirmar

Produtos Stripe a criar:
- surfyng_pro_monthly: R$ 19,90/mês (Price ID: configurar no .env)
- surfyng_global_monthly: R$ 39,90/mês (Price ID: configurar no .env)

   - pm2 save
7. Configurar Nginx como reverse proxy:
   - api.surfyng.com → localhost:3000
8. SSL com Certbot (Let's Encrypt):
   - certbot --nginx -d api.surfyng.com
